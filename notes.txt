Video 10: Create and Visualize a Robotic Arm with URDF
URDF: Universial Robot Description Format, usually a xml file
* Describes robot appearance
* Enables software integration
* Defines physical structure
* Supports simulation capabilities

The body of a robot consists of *links* and *joints* where,
* Links: The rigit pieces of the robot, like bones.
* Joints: Connect links, some can move, enabling motion between connected links.

Within the src folder of the workspace, create the folder which will hold the URDF file.

CD into this folder and create the meta package folder:
  ros2 pkg create --build-type ament_cmake --license BSD-3-Clause <package name>

Use the same command to make the robot description package (in the base package):
  ros2 pkg create --build-type ament_cmake --license BSD-3-Clause <robot name>_description

To check if the newly created packages are recognized by the ROS 2 system:
  source ~/.bashrc
It should do nothing.

Make sure the main package and description package are listed in:
  ros2 pkg list

Make a folder to store the meshes in the description package:
  mkdir -p ~/ros2_ws/src/mycobot_ros2/mycobot_description/meshes/mycobot_280/visual

Make the folders for the urdf files of the different components:
  mkdir -p ~/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/control
  mkdir -p ~/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/mech
  mkdir -p ~/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/sensors
  mkdir -p ~/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/robots

Mesh files are what make your robot look realisting in robotic simulation and visualization programs.
Mesh files are usually in the .stl and .dae formats.

To create custom mesh files, you can use CAD programs like onshape, fusion 360, solidworks, AutoCAD, etc. 
and export them as mesh files in the above listed formats.
You can use Blender to make the mesh file more lightweight and simplified.

For this project we are using pre-made mesh files from Elephant Robotics.
First CD into the downloads folder, then clone the git repo from Automatic Addison:
  git clone -b jazzy https://github.com/automaticaddison/mycobot_ros2.git

Copy the mesh files from the downloaded project into your project (Still in the downloads folder in terminal):
  cp -r mycobot_ros2/mycobot_description/meshes/* ~/ros2_ws/mycobot_ros2/mycobot_description/meshes

In the ros2_ws folder, to install the necessary dependencies from the package.xml:
  rosdep install -i --from-path src --rosdistro $ROS_DISTRO -y

Then build the project.

XACRO is like an architect drawin up plans and URDF is like the final ready to use documents.
Both represent the robot, but XACRO is more flexible. In order to use a XACRO file by ROS 2, it needs
to first be converted into a URDF file.

Create new xacro files:
  touch ~/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/mech/{g_shape_base_v2_0.urdf.xacro,adaptive_gripper.urdf.xacro,mycobot_280_arm.urdf.xacro} ~/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/robots/mycobot_280.urdf.xacro

** BE VERY CAREFUL WITH SPACING IN COMMANDS

To learn basics of XACRO:
  https://docs.ros.org/en/foxy/Tutorials/Intermediate/URDF/Using-Xacro-to-Clean-Up-a-URDF-File.html

For the XACRO files:
* <mimic>: If the mimiced joint moves, the mimicing joint also moves, with a multiplier and offset for the movement.
* <axis>: Defines which direction the joint rotates.
* <limit>: Defines how far (lower and upper bound) a joint can move in radians and how fast (in radians/second).
* <parent> / <child>: Shows the relationship of the joint to attached links.
* <origin>: Where the joint is located relative to it's parent.

<inerital> contains the location of the center of mass of a component, actual mass of compoent, and inertia vector values.

The <box> size dimension is in meters.

joint_effort: Maximum force the joint can exert.
joint_velocity (radians/s): Maximum velocity the joint can move at.

Flange: the connection between main arm component and end effector.

In XACRO file this is the basic flow:
1. Constant definition
2. Helpful macro definitions / implementations
3. Links
4. Joints
5. Gazebo visual properties

* Fixed joint: No relative motion between links.
* Revolute joint: Rotational motion between links.

To convert the XACRO to URDF and visualize it in RIVZ:
  ros2 launch urdf_tutorial display.launch.py model:=/home/<admin name>/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/robots/mycobot_280.urdf.xacro
* Here, replace <admin name> with the name of the user account currently being used for Ubuntu.

To create a pdf map of the connections on the robot, cd into Documents and run:
  ros2 run tf2_tools view_frames

Then to see the pdf you can either open it directly or use the command:
  evince <pdf name>
* <pdf name>: Name of pdf containing the newly generated frame data.

To create an alias to more quickly launch the robot visualization:
  echo "alias elephant='ros2 launch urdf_tutorial display.launch.py model:=/home/<admin name>/ros2_ws/src/mycobot_ros2/mycobot_description/urdf/robots/mycobot_280.urdf.xacro'" >> ~/.bashrc

To find all available aliases:
  alias

If newly made alias is not found, refresh the terminal or:
  source ~/.bashrc
